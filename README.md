TPC-DS Benchmark for MobilityDB
==================================

<img src="doc/images/mobilitydb-logo.svg" width="200" alt="MobilityDB Logo" />

[MobilityDB](https://github.com/ULB-CoDE-WIT/MobilityDB) is an open source software program that adds support for temporal and spatio-temporal objects to the [PostgreSQL](https://www.postgresql.org/) database and its spatial extension [PostGIS](http://postgis.net/).

This repository contains code and the documentation for running an excerpt of the [TPC-DS bechmark](https://www.tpc.org/tpcds/) for analyzing alternative implementations of a temporal data warehouse to assess the performance of temporal algebra and temporal OLAP queries.

The three implementations are
*  The traditional Kimball's Slowly Changing Dimension (SCD) implementation

*  A temporal data warehouse (TDW) implementation proposed by Ahmed et al. in
   the following [article](https://www.igi-global.com/gateway/article/265260)

* A MobilityDB (MobDB) implementation 

Running the benchmark
---------------------

To run the benchmark you must first install the MobilityDB extension in your system. We refer to the [MobilityDB](https://github.com/MobilityDB/MobilityDB) documentation for doing this.

This benchmark uses data generated by the TPC-DS benchmark corresponding to three scale factors (SF): SF1, SF10, and SF100. These data are available in zip format in the files [tpcds_sf1.zip](https://docs.mobilitydb.com/pub/tpcds_sf1.zip) (135M), [tpcds_sf10.zip](https://docs.mobilitydb.com/pub/tpcds_sf10.zip) (1.35G), and [tpcds_sf100.zip](https://docs.mobilitydb.com/pub/tpcds_sf100.zip) (13.5G).

The CSV files are supposed to be in a subdirectory per scale factor under the current subdirectory, e.g., `/home/user/src/MobilityDB-TPCS/sf1/`.
